实施思路

  • ✓前端负责上传、列表、预览、编辑体验；后端负责文件读写和 URL 映射。
  • ✓编辑器建议用 Tiptap/ProseMirror，原型用 contenteditable 模拟直接改字和自动保存。
  • ✓Docker 可挂载多个目录，后端持续监听新增和更新，并写入 SQLite 元数据。
